LBI: The Loolia Brand Index

  1. Home

  3. LBI

Transforming the LBI from a week-long task into a two-click job.

  • Software Engineer
  • Individual
  • 12 months
  • Loolia

This project involves automating the calculation of the Loolia Brand Index (LBI), a manual and time-consuming process. This involves building two separate applications: the main Loolia calculator and a product entry application for logging product appearances in videos.

The Solution

The solution aims to save time, increase accuracy, and eliminate errors in the LBI calculation process. It also includes provisions for future extensions and adjustments to the index.

The implementation consists of two independent applications. The main Loolia calculator is the core component of your project. Its primary purpose is to compute and display the LBI (Loolia Brand Index) for all categories at multiple dates.

The other application is a data entry portal used by the production team to log product appearances on videos in order to source the necessary data to the main application.

Overall, these two applications work in tandem to streamline the LBI calculation process, making it more efficient, accurate, and adaptable to future changes. This automation not only saves time but also enhances the overall quality and reliability of the LBI rankings, ultimately benefiting Loolia's brand promotion and sales efforts.

a manual process that would take over a week to a collective effort that requires a few clicks.

My Responsabilities

As a software engineer, my role encompasses the development of both applications using Java, and for the user interface, I'm leveraging Java Swing. I'm also responsible for the database design and its seamless integration into the applications via JDBC. Managing project dependencies using Maven and Gradle is part of my duties, and I'm ensuring the project's architecture is well-structured.

Challenges

I have encountered two significant challenges in this project:

1. Incorporating Retention Rates

Incorporating retention rates into the Loolia Brand Index (LBI) calculation has proven to be a complex task. Retention rates measure the percentage of users who continue to watch a product video over a specific period, providing valuable insights into product popularity and long-term campaign success. However, several factors have made this challenge noteworthy:

  • Specialized YouTube API
    To access the required level of granularity in retention rate data, a specialized YouTube API setup was necessary. This involved addressing security concerns and handling a high volume of queries.
    Another major hurdle in this phase was Google's policy changes around API access and privacy, which required adapting to their API v3 mid-way through the project.
  • Complexity Increase
    Incorporating retention rates significantly increased the complexity of the LBI calculation process. This complexity involved not only data retrieval but also intricate operations to integrate this data into the index.
  • Computational Demands
    The increased computational complexity also had an impact on system performance. Despite code optimization in Java and query optimization in SQL, we encountered limitations due to the API's latency and the low specifications of the machine running the code. Considering the infrequent usage of this specific part of the code and its ability to run during off-peak times, we decided not to invest additional time in resolving a non-urgent issue.

2. Encourage Adoption

Convincing team members, particularly video editors, to actively participate in the data entry process presented a unique set of challenges. The new workflow required them to log product details and timestamps in the videos they produced. To address this challenge, I approached it with two key considerations:

  • Platform Independence
    Ensuring that the applications are platform-independent, capable of seamless operation on all operating systems, was crucial. Different operating systems come with diverse architectures, libraries, and dependencies, making this a resource-intensive and testing-intensive aspect of the project.
  • User-Friendly Interface
    The success of the application hinged on creating a user-friendly interface that reduced friction in the data entry process. Achieving this required a deep understanding of human-computer interaction, usability engineering, and design thinking. Designing an intuitive and user-friendly interface demanded creativity and expertise in this area.

These challenges, while demanding, have been essential in shaping the project's success. By addressing them effectively, we aim to streamline the LBI calculation process and foster greater user adoption, ultimately enhancing the project's overall impact.
